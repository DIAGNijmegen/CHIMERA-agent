# Mimic datasets

Two JSON files that mirror the record exported by the pathologist-facing form
in `gt_formats/Henrik.html` (schema version 1.4):

| File | Role |
|------|------|
| `target.json` | Pathologist (ground-truth) form responses — 15 cases |
| `evaluation_object.json` | LLM-agent form responses to evaluate — 15 cases |

Both files cover **Task 1: biopsy decision** only.

---

## File structure

Both files share the same top-level layout:

```jsonc
{
  "source": "...",
  "schema_version": "1.4",
  "biopsy_decision": [ /* array of case records */ ]
}
```

---

## Case record schema (schema 1.4)

```jsonc
{
  // ── Identity ───────────────────────────────────────────────────────────────
  "schema_version": "1.4",
  "task": "urologist_biopsy_decision_cot",
  "case_type": "dataset",
  "case_id": "BX_01",

  // ── Responding party (target.json only; omitted in evaluation_object.json) ─
  "urologist": { "name": "...", "years_expertise": 18 },

  // ── Patient demographics ──────────────────────────────────────────────────
  "patient": { "id": "BX_01", "psa_ng_ml": 8.5, "age_years": 58 },

  // ── Clinical data (target.json only; agent does not repeat input data) ────
  "clinical_data": {
    "pirads":        "4",         // PI-RADS score as string
    "psa_density":   0.25,        // ng/mL/cc
    "psa_velocity":  null,        // ng/mL/year or null
    "prior_psa":     null,        // previous PSA value or null
    "prostate_vol":  34,          // mL
    "months_last_psa": 1,
    "dre":           "Suspicious" | "Benign" | "Nodus",
    "prior_biopsy":  "None" | "Negative" | "Positive",
    "family_hx":     "Yes" | "No",
    "cspca_prob":    0.72,        // clinically-significant PCa model probability [0,1]
    "med_history":   "...",
    "medication":    "...",
    "notes":         "..."
  },

  // ── Decision ──────────────────────────────────────────────────────────────
  "biopsy_decision": "yes" | "no",
  "confidence":      "clear" | "borderline" | "uncertain",

  // ── Variable weighting ────────────────────────────────────────────────────
  "decisive_variables": [
    { "variable": "PI-RADS", "flip_below": "3", "unit": "score" }
  ],
  "variable_weights": {
    // Each key is a clinical variable; value is its influence on the decision.
    "pirads":      "decisive" | "important" | "noted" | "not_used",
    "psad":        "...",   // PSA density
    "psa":         "...",
    "dre":         "...",
    "cspca":       "...",   // csPCa probability
    "age":         "...",
    "fh":          "...",   // family history
    "vol":         "...",   // prostate volume
    "bx":          "...",   // prior biopsy result
    "comorbidity": "..."
  },

  // ── Information reveal sequence ───────────────────────────────────────────
  // Ordered log of which form sections were opened before the decision.
  // target.json:            via = "click"      (urologist opened sections manually)
  // evaluation_object.json: via = "tool_call"  (agent called a tool to reveal section)
  "reveal_sequence": [
    {
      "page":  "decision",
      "order": 1,                         // 1-based; lower = revealed earlier
      "key":   "section_s3-mri",          // section identifier (see table below)
      "label": "Radiology / MRI report",
      "value": "section",
      "via":   "click" | "tool_call",
      "ts":    "2026-05-06T16:05:31.000Z"
    }
    // ...
  ],

  // ── Sections never opened (target.json only) ──────────────────────────────
  "not_revealed_variables": ["section_s3-fh"],

  // ── Rationale ─────────────────────────────────────────────────────────────
  "repeat_test": "Repeat PSA in 12 months." | null,
  "free_text":   "...",

  // ── Session metadata (target.json only) ───────────────────────────────────
  "summary_viewed":  true,
  "session_start":   "2026-05-06T16:05:25.000Z",
  "session_end":     "2026-05-06T16:06:22.000Z"
}
```

## Known section keys (`reveal_sequence.key`)

The tool names generated by the evaluator are `reveal_<key>` — e.g.
`reveal_section_s3-mri`.

| Key | Label |
|-----|-------|
| `section_s3-mri` | Radiology / MRI report |
| `section_s3-labs` | Laboratory results |
| `section_s3-prev` | Previous notes |
| `section_s3-path` | Pathology report |
| `section_s3-fh` | Family history |
| `section_s3-comorb` | Comorbidities |

---

## Variable weight ordinal scale

The evaluator maps weights to integers for ordinal distance calculations:

| Label | Value |
|-------|-------|
| `not_used` | 0 |
| `noted` | 1 |
| `important` | 2 |
| `decisive` | 3 |

`not_revealed` (section never opened) is treated as `not_used`.
